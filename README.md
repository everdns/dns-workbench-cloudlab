# DNS Workbench for CloudLab

A CloudLab experiment profile for deploying and testing DNS resolver and authoritative name server software. Supports BIND, PowerDNS, Knot DNS, Knot Resolver, NSD, and Unbound.

All scripts referenced below are located at `/local/repository/` by default on CloudLab nodes.

## CloudLab Profile (profile.py)

The `profile.py` script defines the CloudLab experiment topology. When instantiating the experiment, you can configure the following parameters:

| Parameter | Description | Options |
|---|---|---|
| **Number of Test VMs** | How many test host nodes to create | 1-200 |
| **Separate AS Interface** | Use a dedicated link between the Resolver and Name Server instead of the shared LAN | True / False |
| **Resolver Software** | DNS resolver software to install on the Resolver node | `bind`, `powerdns-recursor`, `unbound`, `knot-resolver`, `all`, `none` |
| **Name Server Software** | Authoritative DNS software to install on the Name Server node | `bind`, `powerdns-authoritative-server`, `knotdns`, `nsd`, `all`, `none` |
| **Allow Interswitch Links** | Allow links that span multiple switches | True / False |

The profile creates three types of nodes:

- **Resolver** -- Runs the selected recursive resolver software. Listens on ports 53 (DNS), 853 (DoT), and 443 (DoH).
- **NS_Local** -- Runs the selected authoritative name server software. Same firewall rules as the Resolver.
- **test_host_N** -- One or more load-testing nodes with testing tools pre-installed.

When a specific software is selected, it is installed and started automatically. When `all` is selected, all supported software for that role is installed but **not** started, allowing you to start the desired one manually.

## start_dns_service.sh

Starts a DNS service on the current node.

```
sudo /local/repository/start_dns_service.sh <software>
```

Available options:

| Option | Description |
|---|---|
| `bind-resolver` | BIND as a recursive resolver |
| `bind-ns` | BIND as an authoritative name server |
| `powerdns-resolver` | PowerDNS Recursor |
| `powerdns-ns` | PowerDNS Authoritative Server |
| `knot-resolver` | Knot Resolver |
| `knot-ns` | Knot DNS (authoritative) |
| `nsd-ns` | NSD (authoritative) |
| `unbound-resolver` | Unbound resolver |

## stop_dns_service.sh

Stops DNS services on the current node.

```
# Stop a specific service
sudo /local/repository/stop_dns_service.sh <software>

# Stop all running DNS services
sudo /local/repository/stop_dns_service.sh
```

Accepts the same software options as `start_dns_service.sh`. When run with no arguments, it stops all DNS services on the node.

## collect_configs.sh

Collects all DNS configuration files from the current node into a single Markdown file.

```
/local/repository/collect_configs.sh
```

- Output is written to `~/dns_configs.md`.
- Checks the configuration directories for each supported software (`/etc/bind`, `/etc/knot`, `/etc/knot-resolver`, `/etc/powerdns`, `/etc/nsd`, `/usr/local/etc/unbound`).
- Zone files (files matching `db.*`) are truncated to the first 15 lines; all other config files are included in full.

## collect_startup_logs.sh

Collects the CloudLab startup execution logs from the current node into a single Markdown file.

```
/local/repository/collect_startup_logs.sh
```

- Output is written to `~/startup_logs.md`.
- Each startup step is recorded with its command, exit status, and full log output.
- Reads from `/var/tmp/startup-N.txt` and `/var/tmp/startup-N.status` files generated by CloudLab's geni startup system.

## uninstall.sh

Removes all DNS software and associated files from the current node.

```
sudo /local/repository/uninstall.sh
```

This script:

1. Stops all running DNS services.
2. Removes NSD and Unbound (compiled from source) via `make uninstall` and deletes their source/config directories.
3. Removes BIND, PowerDNS, Knot DNS, and Knot Resolver via `apt-get remove --purge`.
